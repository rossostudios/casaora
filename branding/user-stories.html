These stories form a living document—refine via user interviews in Paraguay. --- **Comprehensive User Story Catalog for Casaora: Short-Term Rental Property Management & Marketplace Platform (Rebranded from Puerta Abierta)** Casaora builds on a production-ready Supabase + Rust/Axum + Next.js scaffold to deliver an all-in-one solution tailored for Paraguayan property managers, owners, and investors. The database schema enforces multi-tenancy via `organization_id` on nearly every table, with RLS policies ensuring secure access (e.g., `is_org_member(organization_id)`). Key enums support local realities: `expense_category` includes tax/IVA, `reservation_status` tracks full lifecycles, `channel_kind` covers Airbnb/Booking.com/direct, and additional modules for leases, applications, maintenance, workflows, AI chats, and subscriptions enable both short-term rental (STR) excellence and future long-term/hybrid expansion. This catalog expands every possible user story from the existing implementation (properties, reservations, tasks, expenses, owner_statements, message_templates/logs, calendar_blocks, guests, organizations, etc.) plus planned competitive and localized features (AI messaging, real-time channel sync, dynamic pricing, guest/self-service portal, turnkey marketplace like Padsplit, local payments with PYG/USD + 5% IVA handling under Law 1,295, bilingual Spanish-primary UX, and compliance tools). Stories follow the standard format: **As a [persona], I want [feature] so that [benefit]**. Each epic includes a markdown table with columns: **Story ID**, **User Story**, **Acceptance Criteria Highlights**, **Related Schema Tables**, **Phase**, **Priority**. Total exceeds 150 stories for exhaustive coverage. Stakeholders can use this directly for backlog grooming, sprint planning, and design reviews. ### 1.
